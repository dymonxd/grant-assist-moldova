Then create `src/app/actions/auth.ts` with three server actions:

**signup(formData: FormData):**
1. Extract name, email, phone, password, notifications (checkbox), redirectTo from FormData
2. Validate: name required (min 2 chars), email required, password required (min 6 chars) -- return `{error}` in Romanian on validation failure
3. Call `supabase.auth.signUp({ email, password, options: { data: { name, phone }, emailRedirectTo: \`\${process.env.NEXT_PUBLIC_SITE_URL}/api/auth/callback?next=\${redirectTo || '/'}\` } })`
4. If signUp error, return `{ error: error.message }`
5. If session.companyProfileId exists AND data.user exists:
   a. Call `admin.rpc('claim_company_profile', { p_profile_id: session.companyProfileId, p_user_id: data.user.id })`
   b. Call `admin.from('profiles').update({ phone, email_notifications: wantsNotifications }).eq('id', data.user.id)` -- the handle_new_user trigger does NOT extract phone, so we must set it here
   c. Clear session: `session.companyProfileId = undefined; await session.save()`
6. `revalidatePath('/', 'layout')`
7. `redirect(redirectTo || '/')`

**signIn(formData: FormData):**
1. Extract email, password, redirectTo from FormData
2. Call `supabase.auth.signInWithPassword({ email, password })`
3. If error, return `{ error: 'Email sau parola incorecta' }`
4. `revalidatePath('/', 'layout')`
5. `redirect(redirectTo || '/')`

**signOut():**
1. Call `supabase.auth.signOut()`
2. `revalidatePath('/', 'layout')`
3. `redirect('/')`

IMPORTANT: Use `'use server'` directive at top. Follow existing pattern from `src/app/actions/profile.ts` -- return `{error: string}` on failure, Romanian error messages, admin client for privileged operations.

PITFALL: The `redirect()` call from `next/navigation` throws internally (NEXT_REDIRECT error). In tests, mock redirect to throw a custom error and catch it to verify it was called with the right path.

PITFALL: `onAuthStateChange` does NOT fire after server action signup. That is why we use `revalidatePath('/', 'layout')` + `redirect()` to force a full server render.

Create `src/app/actions/__tests__/saved-grants.test.ts` first with tests mocking `@/lib/supabase/server` (createClient with auth.getUser and from().select/insert/delete chain).

Then create `src/app/actions/saved-grants.ts` with two server actions:

**toggleSavedGrant(grantId: string):**
1. Get authenticated user via `supabase.auth.getUser()` -- NOT getSession()
2. If no user, return `{ error: 'Trebuie sa fii autentificat' }`
3. Check if already saved: `supabase.from('saved_grants').select('id').eq('user_id', user.id).eq('grant_id', grantId).maybeSingle()`
4. If existing: delete it, return `{ saved: false }`
5. If not existing: insert `{ user_id: user.id, grant_id: grantId }`, return `{ saved: true }`
6. Call `revalidatePath` for the current page after toggle

NOTE: Use the authenticated server client (createClient), NOT admin client. The saved_grants table has RLS requiring `auth.uid() = user_id`, which validates ownership automatically when using the authenticated client.

**getSavedGrants():**
1. Get authenticated user via `supabase.auth.getUser()`
2. If no user, return `{ grants: [] }`
3. Query: `supabase.from('saved_grants').select('grant_id').eq('user_id', user.id)`
4. Return `{ grants: data.map(d => d.grant_id) }`
