  <behavior>
    - All cron routes return 401 if Authorization header missing or CRON_SECRET mismatch
    - All cron routes return 200 with JSON summary on success
    - Deadline cron: finds grants with deadlines 7 or 3 days from today, finds users with applications for those grants, checks email_notifications=true, checks duplicate within 24h, sends email, logs to notifications_log
    - Deadline cron: skips users who already received a reminder for same grant+type in last 24h
    - Abandoned cron: finds applications with status='in_progress' and updated_at older than 7 days, checks email_notifications, checks duplicate, sends nudge email, logs
    - Abandoned cron: skips users who already received abandoned_draft for same application in last 24h
    - Analytics cron: queries analytics_events for previous day, groups by event_type (mapped to stage) and device_type, counts per group, aggregates top 5 referrers per stage, upserts into analytics_daily_summary
    - Analytics cron: handles days with zero events gracefully (no errors, no inserts)
  </behavior>
  <action>
    1. Create shared cron auth helper (inline in each route or a small shared module):
       ```typescript
       function validateCronSecret(request: Request): boolean {
         const authHeader = request.headers.get('authorization')
         if (!authHeader) return false
         const token = authHeader.replace('Bearer ', '')
         return token === process.env.CRON_SECRET
       }
       ```

    2. Create `src/app/api/cron/deadlines/route.ts`:
       - Export GET handler. Validate CRON_SECRET. If invalid, return Response 401 JSON { error: 'Unauthorized' }.
       - Use createAdminClient (service role for cross-table queries).
       - Query grants WHERE status='active' AND (deadline = CURRENT_DATE + 7 OR deadline = CURRENT_DATE + 3). This gives grants expiring in exactly 7 or 3 days.
       - For each grant: query applications WHERE grant_id = grant.id AND status != 'exported', join profiles for email and email_notifications.
       - For each user/grant pair: skip if email_notifications=false. Call checkDuplicateNotification(userId, grantId, 'deadline_reminder') -- skip if true.
       - Calculate daysLeft from grant.deadline - today.
       - Build email: buildDeadlineReminderEmail({ grantName, deadline, daysLeft, ctaUrl: `/grants/${grantId}/write`, unsubscribeToken }).
       - Send via Resend. Log to notifications_log (type='deadline_reminder', channel='email', grant_id, user_id, unsubscribe_token=gen_random_uuid generated client-side with crypto.randomUUID()).
       - Return Response 200 JSON { sent: N, skipped: M }.

    3. Create `src/app/api/cron/abandoned/route.ts`:
       - Export GET handler. Validate CRON_SECRET.
       - Query applications WHERE status='in_progress' AND updated_at < NOW() - INTERVAL '7 days'.
       - Join profiles for email and email_notifications. Join grants for grant name.
       - For each: skip if email_notifications=false. checkDuplicateNotification(userId, null, 'abandoned_draft') or (userId, grantId, 'abandoned_draft') -- skip if true.
       - Build email: buildAbandonedDraftEmail({ grantName, lastActivity: application.updated_at, ctaUrl: `/grants/${grantId}/write`, unsubscribeToken }).
       - Send via Resend. Log to notifications_log (type='abandoned_draft').
       - Return Response 200 JSON { sent: N, skipped: M }.

    4. Create `src/app/api/cron/analytics/route.ts`:
       - Export GET handler. Validate CRON_SECRET.
       - Calculate yesterday: new Date(); yesterday.setDate(d.getDate()-1); format as YYYY-MM-DD.
       - Query analytics_events WHERE created_at >= yesterday 00:00 AND created_at < today 00:00.
       - Map event_type to stage: { session_start: 'sessions', idno_entered: 'idno_entered', grants_viewed: 'grants_viewed', account_created: 'account_created', writer_started: 'writer_started', application_exported: 'exported' }. Skip unmapped event types (section_generated, section_saved, etc. are not funnel stages).
       - Group by (stage, device_type). Count per group.
       - For each stage: aggregate referrer_url counts across all events, take top 5 as JSONB array.
       - Upsert into analytics_daily_summary using `ON CONFLICT (date, stage, device_type) DO UPDATE SET count = EXCLUDED.count, top_referrers = EXCLUDED.top_referrers`. Use createAdminClient.rpc or raw SQL via .from().upsert() with onConflict.
       - Handle COALESCE for nullable device_type (use 'unknown' as default, matching the existing composite PK pattern from Phase 1).
       - Return Response 200 JSON { date: yesterday, stages_aggregated: N }.

       - Test: abandoned route skips users with email_notifications=false
       - Test: analytics route aggregates events by stage and device_type
       - Test: analytics route handles empty event set gracefully
       - Mock Resend, createAdminClient, checkDuplicateNotification
