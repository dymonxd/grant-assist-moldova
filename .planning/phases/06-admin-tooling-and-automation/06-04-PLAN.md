    - GET /api/unsubscribe with invalid token returns 400
    - getNotificationLog returns notifications joined with profiles and grants, ordered by sent_at DESC
    - getNotificationLog with typeFilter filters by notification type

    - bulkSendNotifications checks email_notifications, checks duplicates, sends emails, logs each send

  <action>
    1. Create `src/lib/email/notification-emails.ts`:
       - Helper: `escapeHtml(text: string)` (same as in application-email.tsx).
       - Helper: `buildUnsubscribeUrl(token: string)`: Returns `${process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000'}/api/unsubscribe?token=${token}`.
       - `buildDeadlineReminderEmail({ grantName, deadline, daysLeft, ctaUrl, unsubscribeToken })`: Inline-style HTML template. Subject line helper: `getDeadlineReminderSubject(grantName, daysLeft)` returns "Memento: {grantName} - {daysLeft} zile ramase". Body: heading with grant name, paragraph "Termenul limita pentru {grantName} este {deadline formatted ro-MD}. Mai aveti {daysLeft} zile.", CTA button "Continua cererea", unsubscribe link at bottom "Dezabonare de la notificari".
       - `buildAbandonedDraftEmail({ grantName, lastActivity, ctaUrl, unsubscribeToken })`: Subject: "Cererea ta pentru {grantName} asteapta". Body: "Ultima activitate: {lastActivity formatted}. Continua de unde ai ramas.", CTA "Reia cererea", unsubscribe.
       - `buildGrantExpiringEmail({ grantName, deadline, ctaUrl, unsubscribeToken })`: Subject: "{grantName} expira curand". Body: deadline info, CTA "Aplica acum", unsubscribe.
       - `buildNewGrantMatchEmail({ grantName, maxFunding, ctaUrl, unsubscribeToken })`: Subject: "Grant nou potrivit: {grantName}". Body: grant name, funding amount formatted, CTA "Vizualizeaza grantul", unsubscribe.
       - All templates use the same base HTML structure as application-email.tsx (max-width 600px, inline styles, sans-serif).

    2. Create `src/lib/email/unsubscribe.ts`:
       - `findUserByUnsubscribeToken(token: string)`: Uses createAdminClient, queries notifications_log where unsubscribe_token=token, returns user_id or null.
       - `disableEmailNotifications(userId: string)`: Uses createAdminClient, updates profiles SET email_notifications=false WHERE id=userId.

    3. Create `src/app/api/unsubscribe/route.ts`:
       - GET handler: Extract token from searchParams. Validate UUID format. Call findUserByUnsubscribeToken. If not found, return Response with 400 and "Link invalid". Call disableEmailNotifications. Redirect to `/settings?unsubscribed=true` (301).

    4. Create `src/app/actions/admin-notifications.ts` with `'use server'`:
       - `getNotificationLog(typeFilter?: string)`: Verify admin. Query notifications_log joined with profiles (name, email) and grants (name). If typeFilter provided, filter by type. Order by sent_at DESC. Limit 200. Return array.
       - `checkDuplicateNotification(userId: string, grantId: string | null, type: string): Promise<boolean>`: Query notifications_log WHERE user_id AND (grant_id or null) AND type AND sent_at > now() - 24 hours. Return exists.
       - `bulkSendNotifications({ type, grantId?, message? })`:
         - Verify admin. Determine target users based on type:
           - deadline_reminder: users with applications for grants expiring in 7 or 3 days, email_notifications=true
           - abandoned_draft: users with in_progress applications updated > 7 days ago, email_notifications=true
           - grant_expiring: users with email_notifications=true (general announcement)
           - new_grant_match: users with matching profiles for specific grantId, email_notifications=true
         - For each user: check email_notifications, check duplicate, build email template, send via Resend, log to notifications_log with unsubscribe_token.
         - Return { sent: number, skipped: number, errors: number }.
       - `getBulkSendPreview({ type, grantId? })`: Verify admin. Run the same targeting query but only return count and sample (first 5 recipients). For confirmation dialog.

    5. Tests in `src/app/actions/__tests__/admin-notifications.test.ts`:
       - Test getNotificationLog returns joined data sorted by sent_at DESC
       - Test getNotificationLog with typeFilter filters correctly
       - Test checkDuplicateNotification returns true for recent duplicate, false for no match
       - Test bulkSendNotifications skips users with email_notifications=false
       - Test bulkSendNotifications skips duplicate notifications (within 24h)
       - Test getBulkSendPreview returns count and sample recipients
       - Test email template builders produce HTML with unsubscribe link
