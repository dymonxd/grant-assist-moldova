  <name>Task 1: Grant catalog server actions with publish validation</name>
  <files>
    src/app/actions/admin-grants.ts,

  </files>
  <behavior>
    - getGrantsCatalog returns all grants with name, provider_agency, deadline, status, application count (from applications join), last_scraped_at, computed displayStatus (Draft/Active/Expiring/Expired)
    - updateDeadline(grantId, newDeadline) updates grants.deadline, returns updated grant
    - duplicateGrant(grantId) copies grant + all grant_application_fields, sets status=draft, appends " (copie)" to name
    - deactivateGrant(grantId) sets status=expired
    - publishGrant(grantId) validates: name non-empty, provider_agency non-empty, deadline is future, at least 1 eligibility_rule in JSONB array, at least 1 grant_application_field row; sets status=active if valid, returns { error } with specific Romanian message if invalid
    - updateGrant(grantId, data) updates grant fields, returns updated grant
    - All actions verify caller is admin (same verifyAdmin pattern as admin-analytics)
    - publishGrant with missing name returns error "Numele grantului este obligatoriu"
    - publishGrant with past deadline returns error "Termenul limita trebuie sa fie in viitor"
  </behavior>
  <action>
    1. Create `src/app/actions/admin-grants.ts` with `'use server'` directive.

    2. Import verifyAdmin helper (or duplicate the pattern from admin-analytics: createClient + getUser + profiles.is_admin check). All actions start with admin verification.

    3. `getGrantsCatalog()`:
       - Use createAdminClient (service role for cross-table joins).
       - Query grants with SELECT *, then separately count applications per grant.
       - For each grant, compute displayStatus: if status === 'draft' -> 'Draft', if status === 'expired' OR deadline < now -> 'Expired', if deadline within 14 days -> 'Expiring', else -> 'Active'.
       - Return sorted by deadline ASC (soonest first).

    4. `updateDeadline(grantId: string, newDeadline: string)`:
       - Validate date format. Update grants.deadline. Return updated row.

    5. `duplicateGrant(grantId: string)`:
       - Fetch original grant + its grant_application_fields.
       - Insert new grant with same data but status='draft', name += ' (copie)'.
       - Insert copied fields with new grant_id.
       - Return new grant.

    6. `deactivateGrant(grantId: string)`:
       - Update grants SET status='expired' WHERE id=grantId.
       - Return { success: true }.

    7. `publishGrant(grantId: string)`:
       - Fetch grant by id.
       - Validate: name (non-empty), provider_agency (non-empty), deadline (future date), eligibility_rules (JSONB array length >= 1), grant_application_fields count >= 1.
       - If any check fails, return { error: 'specific Romanian message' }.
       - If all pass, update status='active', return { success: true }.

    8. `updateGrant(grantId: string, data: Partial<GrantUpdate>)`:
       - Update allowed fields: name, provider_agency, description, max_funding, deadline, eligibility_rules, scoring_rubric, required_documents, source_form_url.
       - Return updated grant.

    9. Tests: Mock createAdminClient with chainable pattern. Test each action including publish validation edge cases (missing name, past deadline, no rules, no fields).
  </action>
