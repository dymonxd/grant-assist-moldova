<objective>
Grant onboarding wizard with AI-powered PDF extraction, publish notifications, and Re-scrape wiring.

Purpose: Enable administrators to efficiently onboard new grants by uploading PDF guidelines and having AI automatically extract eligibility rules, scoring rubric, application fields, and required documents -- turning a manual data-entry task into a review-and-publish flow. On publish, matching users are notified. Existing grants can be re-scraped from the catalog table.

Output: Working 3-step wizard at /admin/grants/new with basic info entry, PDF upload with AI extraction, and review/publish step. Re-scrape with SSRF protection wired into the grant catalog table. Publish notification to matching profiles.
</objective>

    - extractGrantFromPdf with empty text returns empty arrays (graceful degradation)
    - validateScrapeUrl(url: string) returns true for whitelisted domains, false for private IPs and unknown domains
    - validateScrapeUrl rejects 127.0.0.1, 10.x.x.x, 192.168.x.x, localhost
    - createGrantWithExtraction inserts grant + extracted fields into DB
    - notifyMatchingProfiles(grantId) finds profiles matching new grant, checks email_notifications, sends emails, logs to notifications_log
  </behavior>
  <action>
    1. Run `npm install pdf-parse` in the project root (needed for PDF text extraction).

    2. Create `src/lib/ai/extract-grant-pdf.ts`:
       - Import { generateText } from 'ai', { Output } from 'ai', { openai } from '@ai-sdk/openai', { z } from 'zod' (zod is a dependency of ai SDK).
       - Define Zod schema for extraction output:
         ```
         const ExtractionSchema = z.object({
           eligibilityRules: z.array(z.string()),
           scoringRubric: z.array(z.object({ criterion: z.string(), weight: z.number() })),
           applicationFields: z.array(z.object({ label: z.string(), type: z.string(), required: z.boolean(), characterLimit: z.number().nullable() })),
           requiredDocuments: z.array(z.string())
         })
         ```
       - `extractGrantFromPdf(pdfText: string)`: If pdfText is empty/whitespace, return all empty arrays. Otherwise call generateText with model: openai('gpt-5.4-nano'), system prompt in Romanian explaining the task (extract grant program details from document text), user prompt with pdfText, output: Output.object({ schema: ExtractionSchema }). Return the structured output.
       - `validateScrapeUrl(url: string): boolean`: Parse with `new URL(url)`. Check hostname against whitelist array: ['odimm.md', 'ucipifad.md', 'aipa.gov.md', 'e-gov.md', 'gov.md', 'mec.gov.md', 'www.odimm.md', 'www.ucipifad.md']. Also reject if hostname resolves to private ranges: check for 'localhost', '127.', '10.', '192.168.', '0.0.0.0', '169.254.'. Return true only if on whitelist and not private.

    3. Add to `src/app/actions/admin-grants.ts`:
       - `createGrantFromWizard(basicInfo, extractedData)`:
         - Verify admin. Insert grant with basicInfo (name, provider_agency, deadline, description, max_funding) + extractedData (eligibility_rules, scoring_rubric, required_documents) into grants table with status='draft'.
         - Insert each extractedData.applicationFields as a row in grant_application_fields with field_order based on array index.
         - Return { success: true, grantId }.
       - `uploadAndExtractPdf(formData: FormData)`:
         - Verify admin. Get file from formData. Validate: file.type === 'application/pdf', file.size <= 20 * 1024 * 1024 (20MB).
         - Extract text: `const pdfParse = (await import('pdf-parse')).default; const { text } = await pdfParse(Buffer.from(await file.arrayBuffer()))`.
         - Call extractGrantFromPdf(text). Return extracted data.
         - On error, return { error: 'Nu am putut extrage datele din PDF' }.
       - `reScrapeGrant(grantId: string)`:
         - Verify admin. Fetch grant's source_form_url. Call validateScrapeUrl. If invalid, return { error: 'URL-ul nu este permis' }.
         - Fetch URL content, extract text. Call extractGrantFromPdf. Update grant with new extracted data. Update last_scraped_at. Return { success: true }.
       - `notifyMatchingProfiles(grantId: string)`:
         - Verify admin. Fetch the new grant. Query company_profiles joined with profiles where email_notifications=true.
         - For each profile, check notifications_log for existing new_grant_match for this grant (prevent duplicates).
         - Send email via Resend with grant name, funding, deadline, CTA link.
         - Log each send to notifications_log with type='new_grant_match'.
         - Return { sent: count }.

    4. Tests in `src/lib/ai/__tests__/extract-grant-pdf.test.ts`:
       - Mock ai module's generateText. Test extractGrantFromPdf returns structured data.
       - Test extractGrantFromPdf with empty text returns empty arrays.
       - Test validateScrapeUrl accepts 'https://odimm.md/grants', rejects 'http://127.0.0.1/secret', rejects 'https://evil.com/exploit'.
       - Test validateScrapeUrl rejects private IP patterns.
